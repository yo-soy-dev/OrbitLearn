## â—ˆ Integration Notes

### Stripe Webhooks â†’ `/api/webhooks/stripe`
```
invoice.paid                   activate / renew subscription
customer.subscription.updated  handle tier changes
customer.subscription.deleted  downgrade or cancel
```
Always verify signatures with `STRIPE_WEBHOOK_SECRET`. Store all Stripe IDs server-side â€” never trust client-reported status.

### Vapi Sessions
Server issues a short-lived access token â†’ client joins via Vapi SDK â†’ `vapi_room_id` persisted in `sessions`. On `session.ended`, webhook triggers OpenAI summary generation â†’ stored in `summaries` â†’ emailed via Brevo.

### AI Endpoints

```
POST /api/ai/generate-quiz
  body:  { lessonText: string, numQuestions: number }
  out:   { questions: [{ question, options, correct_index, explanation }] }

POST /api/ai/generate-summary
  body:  { sessionId: string, transcript: string }
  out:   { summary: string (Markdown) }
```

Use a JSON schema in your OpenAI prompt to enforce structure. Store results before returning â€” don't regenerate on every request.
